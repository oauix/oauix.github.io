常用关键词

1.break:跳出循环

2.continue:跳过循环体中剩余语句而进入下一次循环

3.return：返回数据，或跳出方法

4.final

最终的、不可变的------单独应用几率不高

1)修饰变量:变量不可被改变

2)修饰方法:方法不可被重写

3)修饰类:类不可被继承

5.finally 异常处理，不管是否出现异常，都会执行，常用于结束某些进程


6.static 静态的

1)静态变量:

1.1)由static修饰

1.2)属于类，存在方法区[内存]中，只有一份

1.3)常常通过类名.来访问

1.4)何时用:所有对象共享的数据(图片、音频、视频等)

2)静态方法:

2.1)由static修饰

2.2)属于类，存在方法区中，只有一份

2.3)常常通过类名.来访问

2.4)静态方法没有隐式的this传递，

静态方法中不能直接访问实例成员

案列：

	public class tester{
		public static void main(String[] agrs){//静态，是没有this传递的
			say();//为实例方法，默认是this.访问的，现在没有this传递故会报错
			tester t=new tester();
			t.say();
			sayhi();//默认是类名.sayhi()访问
		}
		int i=8;//实例变量
		static int b=5;//静态变量
		void say(){//实例方法
			System.out.println(i);//默认是this.i
			System.out.println(b);//默认是类名.b
		}
		static void sayhi(){//没有默认this传递静态方法
			System.out.println(i);//默认是this.i 这里没有this.不能访问
			System.out.println(b);//默认是类名.b
		}
	}

	//Math.sqrl();静态方法

2.5)何时用:方法的操作仅与参数相关而与对象无关

3)静态块:

3.1)由static修饰

3.2)类被加载期间自动执行，因类只被加载一次，

所以静态块也只执行一次

3.3)何时用:常常用于初始化静态资源(图片、音频、视频等)

7.static final常量: 应用率高

1)常量必须声明同时初始化

2)通过类名点来访问，不可被改变

3)建议:常量名所有字母都大写，多个单词用_分隔

4)编译器在编译时将常量直接替换为具体的值，效率高

8.this与super

this:指代当前对象，哪个对象调方法指的就是哪个对象

只能用在方法中，方法中访问成员变量之前默认有个this.

this的用法:

1)this.成员变量名--------访问成员变量

2)this.方法名()----------调用方法(不常用)

3)this()-----------------调用构造方法

super:指代当前对象的父类对象
super的用法:

super.成员变量名------------访问父类的成员变量

super.方法名()--------------调用父类的方法

super()---------------------调用父类的构造方法

构造方法是不能被继承的，它是子类构造方法里有一个super()调用

9.abstract 抽象关键词

10.instanceof 向下造型

在强制转型中，为了避免出现ClassCastException[类型转换错误]，可以通过instanceof关键字判断某个引用指向的对象是否为指定类型。（陌陌是否是他的子类）

11.package:

1)避免类名冲突 //作用

2)包名可以有层次结构，同包中的类不能同名

3)类的完全限定名: 包名.类名

4)建议:包名所有字母都小写

包名不能用java关键词，试了用"java"作为包名，JFrame运行报错

12.import:

1)同包中的类可以直接访问，不同包中的类不能直接访问，只有如下两种方式:

1.1)先import声明类，再访问类----建议

1.2)类的完全限定名-------太繁琐，不建议

13.访问控制修饰符[修饰关键词]:

1)public:公开的，任何类

2)private:私有的，本类

3)protected:受保护的，本类、子类、同包类

4)默认的:什么也不写，本类、同包类

类的访问修饰: public和默认的

类中成员访问修饰: 上面4种都可以
