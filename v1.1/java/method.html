方法:

1.方法介绍:

1)用于封装一段特定的业务逻辑功能

2)方法要尽可能的独立，只干一件事

3)方法可以被反复调用多次

4)避免代码重复，有利于代码的维护，有利于团队协作


方法的定义:---表达式

修饰词 返回值类型 方法名(参数列表){

方法体

}

方法的返回值，何时有返回值?何时没有返回值?

方法可以有返回值也可以没有返回值:

无返回值，则返回值类型写成void

有返回值，则返回值类型写成具体的数据类型

何时有返回值?何时没有返回值?

若方法调用结束后，还需要用到方法中的某个数据---有返回值

不需要用到方法中的某个数据---无返回值

return 返回值；

或 方法调用

java方法调用：就近调用原则，方法调用先子后父。

2.方法的重载(Overload):

1)发生在一个类中，方法名称相同，参数列表不同

2)编译器在编译时会根据签名自动绑定调用的方法


构造方法:

1)目的：给成员变量赋初值[我们又称初始化]

2)与类同名，没有返回值类型，并不写void关键词

3)在创建对象时被自动调用

4)若自己不写构造方法，则编译器默认一个无参构造方法,若自己写了构造，则不再默认提供

5)构造方法也可以重载


3.方法的重写(Override):

1)发生在父子类中，方法名称相同，参数列表相同，方法体不同

2)重写方法被调用时，看对象的类型


4.重写与重载的区别:-----常见面试题


1)重写(Override):

1.1)发生在父子类中，方法名相同，参数列表相同，方法体不同

1.2)重写遵循"运行期"绑定，看对象的类型来调用方法


2)重载(Overload):

2.1)发生在一个类中，方法名相同，参数列表不同，方法体不同

2.2)重载遵循"编译期"绑定，看引用的类型[即方法参数]来绑定方法

问:子类可以继承父类的构造方法吗?面试题

答:不可以，父类的构造是被子类调用的


5.抽象方法:

1)由abstract修饰

2)只有方法的定义，没有方法的具体实现(连{}都没有)

3)要用时必须重写

【扩展】在事件监听器中，监听器就接口，但为了用时不用全部重写里面的方法，于是推出了事件适配器，它里面是中实现
