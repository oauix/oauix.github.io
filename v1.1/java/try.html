finally块：

*finally块必须定义在异常处理机制的最后.

* finally块可以保证无论try块中的代码是否出现异常,finally块中的代码都会执行.

* 通常会将诸如释放资源等操作放在finally中确保执行.

throws异常抛出

#可以自己建立异常，提醒别人处理：

	class Poe extends RunTimeException{
		private int age;
		public int getage(int age) throws Exception{ //声明该方法有异常，告知后者处理
			if(age>0 && age<){
				throw new Exception("异常，输入的年龄不合法！！");//建立一个异常
			}
			return age;
		}
	}

#异常不处理，否则编译不通过.而处理手段有两种:

* 1:使用try-catch捕获并处理

* 2:继续在当前方法上使用throws将该异常声明继续向外抛出


RuntimeException 非检测异常[又称常识错误,大家都知道常识就不用检测]

检测异常需要处理，否则编译器不通过，不允许编译，非检测异常就不用

但只是编译时不去检测异常，运行时有异常程序还是会停止运行

常见的异常 [关联异常处理]

引用=null 、NullPointerException

null:空，没有指向任何对象

若引用的值为null，则不能再进行任何操作了

若操作则空指针异常
